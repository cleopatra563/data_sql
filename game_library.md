# 游戏竞品库设计需求文档

## 1. 项目概述

### 1.1 项目背景
游戏行业竞争激烈，及时了解竞争对手的产品动态、功能设计、运营策略对游戏开发团队至关重要。目前团队缺乏一个集中管理和分析游戏竞品信息的平台，导致竞品情报分散、分析效率低下。

### 1.2 项目目标
建立一个全面的游戏竞品库，实现竞品信息的收集、存储、分析和可视化，帮助团队快速获取竞品情报，支持产品决策。

### 1.3 适用范围
适用于游戏发行和开发团队（策划、开发、运营、市场等角色）。

## 2. 功能需求

### 2.1 竞品信息管理
- **竞品获取渠道**：
  - **应用商店**：App Store、Google Play、TapTap
  - **官方渠道**：游戏官网、官方社交媒体账号、官方论坛
  - **第三方数据平台**：Sensor Tower、App Annie、DataEye、AppGrowing
  - **媒体渠道**：游戏媒体（IGN、GameSpot、游资网）、行业博客、新闻网站
  - **社交平台**：Twitter、Facebook、Discord、Reddit
  - **直播平台**：Twitch、YouTube Gaming、斗鱼、虎牙
  - **行业展会**：E3、GDC、ChinaJoy、东京电玩展
  - **玩家社区**：贴吧、NGA、Steam社区、TapTap社区
  - **公开数据**：公司财报、行业报告、研究机构数据
- **游戏基本信息**：名称、类型、发行商、开发商、发行日期、平台（PC/移动端/主机）、投放地区、包名/应用ID、商店五图、下载链接、系统平台、官网链接
- **核心玩法**：玩法描述、核心机制、创新点
- **美术风格**：画面类型（2D/3D）、风格描述、视觉效果
- **技术架构**：引擎、技术亮点
- **运营数据**：下载量、活跃度、收入、评分、内购数据、RPD、次留、7留、30留、CPI、CTR、ecpm
- **榜单数据**：GP/IOS榜单数据（下载量、活跃用户、评分、排名等），付费+免费，国家范围：美国、巴西、印度、印尼、菲律宾、港澳台
- **更新日志**：版本迭代记录、功能更新内容

### 2.2 竞品分析功能
- **对比分析**：支持多竞品横向对比（玩法、美术、技术、运营等维度）
- **分析粒度**：支持汇总与明细数据对比，明细数据包括（周、月、年）
- **趋势分析**：竞品数据变化趋势图表（收入、活跃用户等），按时间维度（周、月、年）分析，按游戏基本信息（类型、发行商、平台等）分析
- **标签管理**：支持自定义标签，便于分类和检索
- **投放策略**：根据投放量划分预约期、冲刺期、稳定期、近期，给出时间范围、系统平台、包含地区，列出主投素材类型、主投地区、主投媒体平台、核心受众、高曝光广告

### 2.3 竞品买量素材
- **素材获取**：
  - 自动抓取：支持从Facebook、Google Ads、TikTok、Unity Ads等主流广告平台自动抓取竞品买量素材
  - 手动上传：支持团队成员手动上传收集到的竞品买量素材
  - 素材类型：图片（单图、多图）、视频、HTML5互动素材
  - 素材元数据：素材ID、广告主、投放平台、投放地区、投放时间、素材尺寸、素材格式

- **素材管理**：
  - 素材分类：按游戏、素材类型、投放平台、投放地区、时间等维度分类
  - 标签管理：支持自定义标签（如"核心玩法展示"、"剧情向"、"活动推广"等），具体见标签字典
  - 素材存储：支持高清素材存储，可预览和下载
  - 素材版本：支持追踪同一素材的不同版本迭代

- **素材分析**：
  - **效果分析**：展示素材的CTR、CVR、CPI、ROI等关键指标
  - **趋势分析**：分析素材投放效果随时间的变化趋势
  - **竞品对比**：支持多竞品买量素材的横向对比（创意方向、投放策略、效果数据）
  - **创意分析**：自动识别素材中的关键元素（角色、场景、文案、BGM等）
  - **脚本分析**：
    - **内容主题**:（内容主题+表现形式+用户心理+转化策略+技术特征）
    - **结构分析**:（开场吸引+核心展示+利益点呈现+行动号召）
  - **热点追踪**：实时追踪行业热门买量素材和创意方向

- **智能推荐**：
  - 识别竞品成功素材的核心元素，为自身买量提供参考
  - 基于投放效果数据，推荐优质素材创意方向（后面做）
  - 预警低效素材模式，避免无效投放（后面做）

### 2.4 海外头部产品营销动作
- **营销动作范围**：覆盖全球主要市场（美国、欧洲、东南亚、中东等）的头部游戏产品
- **营销渠道监测**：
  - 社交媒体（Facebook、Twitter、Instagram、TikTok、Discord）
  - 视频平台（YouTube、Twitch、Bilibili海外版）
  - 游戏媒体（IGN、GameSpot、Polygon、本地游戏媒体）
  - 直播平台（Twitch、YouTube Gaming）
  - 应用商店（App Store、Google Play、本地应用商店）

- **营销内容类型**：
  - **社交媒体内容**：帖子、短视频、直播、互动活动
  - **广告投放**：展示广告、视频广告、原生广告、搜索广告
  - **PR活动**：新闻稿、媒体评测、开发者访谈
  - **社区运营**：论坛活动、Discord社区建设、玩家反馈收集
  - **跨界合作**：IP联动、品牌合作、KOL/KOC合作
  - **线下活动**：游戏展会、线下体验店、玩家见面会

- **营销数据分析**：
  - **效果追踪**：监测营销活动的曝光量、互动量、转化率
  - **趋势分析**：分析不同市场的营销热点和趋势变化
  - **竞品对比**：对比不同竞品的营销策略和效果

- **营销动作预警**：
  - 竞品发布重大营销活动时自动通知
  - 监测到行业营销趋势变化时预警
  - 重要KOL/KOC合作信息推送

- **案例库建设**：
  - 收集整理海外头部产品的成功营销案例
  - 分析案例的策略、执行和效果
  - 建立可检索的案例数据库

### 2.5 行业动态（海外市场）
- **海外大事件监测**：
  - **海外行业展会**：E3、GDC、东京电玩展、科隆游戏展等海外主要展会
  - **海外公司动态**：海外游戏公司并购、IPO、重大合作等
  - **海外游戏新品发布**：海外市场新游上线、版本更新、资料片发布
  - **海外政策法规变化**：海外各国游戏相关政策法规调整
  - **海外技术突破**：海外游戏技术创新、新引擎发布等
  - **监测渠道**：海外应用商店、官网、Twitter、Discord、海外游戏媒体
- **海外买量成本洞察**：分全球、海外地区，分日的买量成本趋势（大盘线）
- **海外更新提醒**：海外竞品有重大更新时自动通知相关人员
- **买量成本洞察**：分全球、地区，分日的买量成本趋势（大盘线）
- **更新提醒**：竞品有重大更新时自动通知相关人员

### 2.6 权限管理
- **角色划分**：管理员、分析师（数分）、普通用户（策划、市场、运营、投放、UE）
- **权限控制**：不同角色拥有不同的操作权限
- **数据可见性**：可设置某些竞品信息仅对特定团队可见

### 2.6 导出与分享
- **数据导出**：支持将竞品信息导出为Excel/PDF格式
- **报告生成**：自动生成竞品分析报告
- **团队分享**：支持内部分享分析结果和评论

## 3. 非功能需求

### 3.1 性能需求
- 页面加载时间 < 2秒
- 数据查询响应时间 < 1秒
- 支持至少100个并发用户

### 3.2 安全性需求
- 数据加密存储
- 访问日志记录
- 定期数据备份

### 3.3 可用性需求
- 系统可用性 > 99.9%
- 支持多浏览器访问（Chrome、Firefox、Safari、Edge）
- 响应式设计，支持移动端访问

### 3.4 扩展性需求
- 支持新游戏平台的扩展
- 支持自定义分析维度和指标

## 6. 界面设计原则

### 6.1 设计风格
- 简洁明了，重点突出数据和分析结果
- 采用蓝色为主色调，体现专业性和可信度
- 响应式设计，适配不同设备

### 6.2 核心页面结构
- 顶部导航栏：搜索、通知、用户中心
- 左侧菜单栏：功能模块导航
- 主内容区：数据展示和操作区域
- 右侧信息栏：快速访问常用功能或统计数据

### 8.1 页面跳转关系

```
┌──────────────────────────────────────────────────────────────────────────────┐
│ 游戏竞品库 - 页面跳转关系                                                     │
└──────────────────────────────────────────────────────────────────────────────┘

首页 → 竞品管理页 → 竞品详情页
   ↓            ↓            ↓
素材分析页 → 投放策略分析页 → 营销动作分析页
   ↓            ↓            ↓
行业动态页 ← 报告导出与分享页 ← 系统管理页

详细跳转流程：
1. 用户登录系统后进入首页，可通过左侧菜单或快速入口访问各功能模块
2. 首页 → 竞品管理页：点击"竞品管理"快速入口或左侧菜单
3. 竞品管理页 → 竞品详情页：点击竞品列表中的"查看"按钮
4. 竞品详情页 → 投放策略分析页：点击"投放策略"标签或按钮
5. 竞品详情页 → 买量素材分析页：点击"买量素材"标签或按钮
6. 竞品详情页 → 营销动作分析页：点击"营销动作"标签或按钮
7. 任何页面 → 竞品对比页：通过顶部导航栏的"对比分析"功能
8. 任何页面 → 报告导出与分享页：通过页面顶部的"导出"或"分享"按钮
9. 管理员角色 → 系统管理页：通过左侧菜单的"系统管理"模块
10. 任何页面 → 首页：点击顶部导航栏的logo或"首页"按钮

交互设计：
- 所有页面都包含顶部导航栏和左侧菜单栏，确保一致的导航体验
- 关键操作提供明确的按钮和链接，如"新增"、"编辑"、"删除"、"导出"等
- 页面间跳转提供加载状态反馈，提升用户体验
- 支持面包屑导航，显示当前页面在系统中的位置
```

## 10. 风险评估

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 竞品数据获取困难 | 数据不完整 | 结合自动爬虫和手动录入，建立多渠道数据来源 |
| 数据更新不及时 | 情报价值降低 | 设置自动更新机制，定期抓取最新数据 |
| 用户使用率低 | 项目价值无法体现 | 提供培训、简化操作流程、增加实用功能 |
| 数据安全性问题 | 敏感信息泄露 | 加密存储、权限控制、访问日志记录 |
